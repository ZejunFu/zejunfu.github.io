十八个棍拐 总算拨云见天明 不过尔尔

莫言此事难在开头 当下不过暂歇 实则中道见异思迁 叫苦连天 噫 小修士何不想当真大仙

序言 ：禅宗一人痴 误入五行山

于家中LAPTOP前参禅打坐。知众人以Python念诀 口里嗫嚅pip install真言。
搭建个人博客的执念始于某个夜晚 之后就是第一次打开阿里云的页面去选购域名和服务器。我之前总习惯可视化软件美化后端给我的网页 面对CMD都要生出激动。做些大概如下的事：下载Hexo和Git 搞得前端框架的组件像未拼合的榫卯(那时我还不很明白什么东西叫“前端”) 第一次用VScode打开终端时给我的反馈是“Git无法被认定为是一个有意义的或可被调用的”。
在终端窗口第一次喷涌出猩红的error时 我的笔记本正在晚上发出算作恼人的嗡鸣——那是我与机器语言的初次对峙。

注：嗡鸣源于我错误的认知 认为建Blog这件事若非打开笔记本的“高性能模式”是不可为的

第一回 ：报错如妖阵 debug似取经

听闻西方有极乐世界名曰GitHub 又晓互联网中一众修士铸就Blog洞府。吾便以阿里云为八卦罗盘 域名备案为通关文牒 想入洞天。却不知前端乃画符之术 后端乃炼器之法 以致初见cmd黑匣 恍若跌入无底深渊 遥望SSL金锁链 恰似误触天庭禁制。Type化执念 code满屏飞 如是远程代仓已连接 便是成道小圆满。

又是再一次npm install hexo-deployer-git –save 草了 这句话我输过无数遍。机械地输入第n遍指令 脑袋自从昨天折腾完Node.js版本时就已宕机 然后就是第七行dependency警告浮现。当时坚持下去的理由主要是我已经下载了这么多东西 有Hexo Git 它们Path的系统环境配置也做了 注册Gitee又发现Page管理的服务被取消掉了又重新在Github建立了仓库 要是放弃 我真不想把这些东西都删掉——我笨成这样 删都删不干净。
主要是凭着”来都来了”精神 我去CSDN找教程 心中发出感叹 怎么最近的教程也要追溯到21年 Blog的时代真是过去了。三天前装主题踩坑时这么说过 两天前被YAML语法折磨时也念叨过。
把ssh-agent的进程ID输进终端 已经改了不少次 按了回车又弹”Permission denied (publickey)”错误提示依旧。

我就想 啥时候hexo clean&&hexo generate&&hexo deploy这三件套能都不报错啊 这时候最大的进步就是明白git不被识别为可用的是因为没用Git Bash打开。当然 最终这些问题我全部解决了 不然这个网页也不会出现。它们的下载调试很大原因是因为我Hexo版本超前Git版本原始 在我更新版本后调试并不困难。

WARNING：
[ 依赖项建立失败 阁下还是重入轮回来过 ]

第二回 ：执念起 满屏飞

忽有一日 Git Bash惊现血红色警示：致命合并冲突。持键盘 Stack Overflow为经 与Permission denied鏖战。更遇SSH密钥化作六耳 分不清本尊泡影。最险时 天机秘法CSS缺了分号 竟让整个洞府坍缩成量子态

配置只是一般的过程 我们总在遇到更困难的挫折前就觉得眼前已是灾难。
故事的开始总是美好的 点开npm start开始本地调试后 蓝色蝴蝶的头像出现 弹出来了初始化的网页 一切都很正常。只是在我着手网页个性化时 something wrong。图片路径填进_config.yml的background项 控制台说YAMLException 后来才知静态资源必须放在主题目录的source文件夹 还要在路径前加上的”/“符号。
按照教程安装hexo-tag-aplayer时 npm突然报错404 在排除所有我可能写错的地方 结论是最新版早已弃用某些依赖。从Github历史版本里刨出v3.0.4勉强装上 却发现歌词滚动效果把文章目录挤成了俄罗斯方块。如果忧郁是一种天赋——其实命苦就是命苦。
把waline评论系统配置了第8遍仍然报错 在Github issue里发现某位开发者留下的禅语：”检查你的avatarCDN参数是否被转义”

阿弥陀佛。

我把这些都搞定了 做出了现在这个我喜欢的网页 浪客行的背景 配图寓意就和我Blog的域名一样：t3p——意为Type To The Peak

（注：此劫后被录入《程序员渡厄手册》 名曰《字符引发的血案》）

第三回 ：真糊涂见真章 假心思获假货

疯言：这些文字在宏观尺度呈现布朗运动 在普朗克尺度却严格遵循泡利不相容原理————每个荒诞比喻都独占特定能级。
然 真正的疯 是给每个荒诞找到求解方程 真正的傻 是把每句疯话当作无意义的解

最后一劫是控制台警告说某个伪元素选择器吃掉了后面的媒体查询，而我甚至分不清哪里该用::before还是:after。更讽刺的是当我准备放弃某些实在不兼容的功能时 发现连撤回修改都会触发cache的诅咒。
不过终于 Blog首页浏览器能打开了——随着控制台输出的绿色success和令人欣喜的Deploy done： git。那些不解到底为何报错的时候 此刻都化作Markdown文档里平静的句读。前辈在代码里留下的注释 开源社区素未谋面的贡献者 都扶持着笨拙的我做出这一点点。

从前报错如梦幻泡影消散 而今文章列表如河图排列
Git Bash须臾现千里江山图 IDE内集成藏奇门遁甲阵

Zejun Fu此刻顿觉：终究逃不过给世间万物赋范的命运。

第四回 ：化梦成真圣 圣真成梦话

回望如同儿戏的寥寥几天 方才懂得编译器报错实则是另类的人生讲义。代码世界没有玄妙卦象 唯有逻辑的齿轮严丝合缝地转动。那些被版本控制系统封存的commit记录 何尝不是数字修行者的功德碑？
好像我习惯觉得自己做出的成果不过尔尔 哪里轮得到“回顾走来的路” 现在对着逼仄的空间我想——或许本来就不该总想着等我取得大成果 大成功再夸夸而谈。对自己来说 再小的成果和历程都值得被记录和总结。这些文字本就不是为了示人讨喜而存在 又何必害臊自谦。
修炼成道的功法世间千万条 又有哪条算得上容易 又有哪条算得上最好？
于你 Zejun Fu来说 能靠自己走通的法子就是最好的法子 而坚持本身 就是最优雅的算法。

结语 ： 小西天封不了真大圣 取真经又哪能找外包

总算是用稀碎的编排写了点浅显的感悟 总说只有内心充满不安全感的人才会想要做任何事之前找到同行者或先例 而内心强大的人是敢于仅凭个人对未知的向往而前进的 希望我也能成为一个敢于独行攀登的人 一个不被别人定义价值的人。
还有 如果哪一天迷茫了 困惑了 就点开看看吧 总有人走在前头 要勇敢地去尾随追赶。 解惑真经
曰：

初羡旁修识符经 今入浅海已似溺
若炼六根成圣道 先入龙宫取棍棒
苦修魔修盖正法 不乱初心得始终
洞天一片躬耕扩 光阴如金几行间

值此洞天完工 特将渡劫心得封入此页：

苦修经：报错时长与成就快感呈指数正相关 四小时debug换来的顿悟 胜过百篇教程
代码禅宗：IDE不是法器而是木鱼 敲击键盘时请默念 若修真经 先鉴真我
